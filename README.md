# 🚀 Doable
### *The Missing Layer Between AI and Execution*

> **Built for the Not Your College (NYC) CodeQuest Hackathon 2026**

---

## 🌍 The Problem & Our Solution
Modern AI is incredible at generating roadmaps and answering questions, but it stops at **the answer**. Users bookmark roadmaps and promise to start tomorrow, but rarely do. 

The gap between knowing and doing is what inspired **Doable**. Instead of acting like another chatbot, Doable transforms every goal into a guided journey with built-in accountability. It doesn't just explain how to do something—it makes sure you actually finish it.

---

## ⚙️ How Doable Works
1. **Describe your goal**: Tell Doable what you want to achieve (e.g., "Build a portfolio").
2. **AI Generates a Roadmap**: The AI creates a structured, step-by-step plan with actionable tasks.
3. **Focus on One Task**: Doable unlocks progress one step at a time to prevent overwhelm.
4. **Submit Proof**: Upload a screenshot or note proving you actually completed the work. No shortcuts.
5. **Build Your World**: Every completed task contributes to building a personalized, interactive 3D virtual world, rewarding your consistency.

---

## ✨ Key Features
- **🤖 AI Workspace**: Generate personalized roadmaps from natural language goals.
- **📍 Smart Planning**: Break large, intimidating goals into small, achievable milestones.
- **📷 Proof Submission**: Upload proof to unlock the next task and build a track record.
- **🌍 Interactive 3D World**: Make your progress visible by growing a virtual environment.
- **📊 Dashboard**: Track your current goal, activity, and growth all from one place.
- **🎨 Theme Support**: Unlock multiple visual themes for a personalized experience.

## 🏗 Tech Stack
- **Frontend**: React 18, TypeScript, Vite
- **Styling**: Tailwind CSS, shadcn/ui, Radix UI
- **Animations**: Framer Motion, React Spring
- **3D Graphics**: Three.js, React Three Fiber, Drei

## 🚀 Running Locally

```bash
git clone https://github.com/AdiTheCoder00/doable.git
cd doable
npm install
npm run dev
```
*(Use `npm run build` and `npm run preview` to test the production build locally).*
